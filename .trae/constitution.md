# 项目章程/约束总则

## 技术选型
- 纯原生 HTML/CSS/JS + ES Modules
- 部署：GitHub Pages（静态托管）
- 用户可配置 MCP/云服务 URL 连接后端 AI
- 数据全存浏览器本地（localStorage + IndexedDB）

## 铁律（不可违）

### 代码
1. 单文件不超过1000行，超过必须拆文件
2. 禁止占位、TODO、假逻辑、假按钮
3. 禁止省代码：该有的分支、循环、存储、联动必须写全
4. 不改无关文件，不顺手优化，不偷偷重构

### 数据
5. 一份数据一个来源，不散写多份
6. 展示层只读，不自己另存一份
7. 配置集中放`data/`和`core/config.js`，不散落在组件里
8. 不写死APP列表、顺序、主题、壁纸、角色名、默认称呼

### 联动
9. APP之间禁止直接互调，必须走事件中心
10. 所有APP事件最终进入inbox，AI从inbox读取
11. 通知走统一判断层，不绕过开关

### 存储
12. localStorage只放轻配置
13. 聊天记录、记忆、大通知强制走IndexedDB
14. 角色相关数据全部带characterId隔离
15. 存储键全部定义在`core/storage-keys.js`，不临时起名

### UI
16. 所有颜色CSS变量，禁止硬编码色值
17. 图标线条风SVG，禁止emoji、实心黑图标
18. 大圆角、柔阴影、同色系、低饱和
19. 动画不超过350ms，禁止linear缓动
20. 禁止纯黑#000文字、灰色rgba(0,0,0,x)阴影

### 文案
21. 自然可爱，禁止干巴巴技术文案
22. 禁止系统播报腔、客服备注腔

## 主题系统
- 4套：日间粉/日间蓝/夜间奶棕/夜间黑粉
- 手动切换，切换结果持久化
- 所有页面走同一套CSS变量
- 字体粗体，颜色棕黑/灰黑/奶白/米白，适配主题

## 角色系统
- 所有角色数据隔离（记忆、聊天、资料、状态）
- 切换角色=切换作用域
- AI记忆第一人称，符合角色人设

## 工作流
1. 接到任务 → 先看现有代码 → 判断影响范围
2. 改代码 → 自检 → 确认联动没断 → 确认没牵连其他APP
3. 遇到不确定的 → 问用户，用大白话说明选项和后果
4. 每阶段完成后核对roadmap

## 文件1000行拆分规则
- 超过1000行必须拆成子文件
- 拆文件不砍功能，不阉割逻辑
- 按单一职责拆分，调用关系自洽
- 用户找问题→找对应文件夹→只改对应文件
