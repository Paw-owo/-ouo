# 项目章程/约束总则

## 技术选型
- 纯原生 HTML/CSS/JS + ES Modules
- 部署：GitHub Pages（静态托管）
- 用户可配置 MCP/云服务 URL 连接后端 AI
- 数据全存浏览器本地（localStorage + IndexedDB）

## 铁律（不可违）

### 代码
1. 单文件不超过800行，超过必须拆文件
2. 禁止占位、TODO、假逻辑、假按钮
3. 禁止省代码：该有的分支、循环、存储、联动必须写全
4. 不改无关文件，不顺手优化，不偷偷重构

### 数据
5. 一份数据一个来源，不散写多份
6. 展示层只读，不自己另存一份
7. 配置集中放 `data/` 和 `core/config.js`，不散落在组件里
8. 不写死APP列表、顺序、主题、壁纸、角色名、默认称呼

### 联动
9. APP之间禁止直接互调，必须走事件中心
10. 所有APP事件最终进入 inbox，AI从 inbox 读取
11. 通知走 `core/notifications.js` 统一判断层，不绕过开关
12. `ai-events.js` 只做筛选/映射，不另建事件存储

### 存储
13. localStorage 只放轻配置
14. 聊天记录、记忆、大通知强制走 IndexedDB
15. 角色相关数据全部带 characterId 隔离
16. 存储键全部定义在 `core/storage-keys.js`，不临时起名

### UI
17. 所有颜色 CSS 变量，禁止硬编码色值
18. 图标线条风 SVG，禁止 emoji、实心黑图标
19. 大圆角、柔阴影、同色系、低饱和
20. 动画不超过 350ms，禁止 linear 缓动
21. 禁止纯黑文字、灰色 rgba(0,0,0,x) 阴影
22. 字体颜色必须走 CSS 变量

### 文案
23. 自然可爱，禁止干巴巴技术文案
24. 禁止系统播报腔、客服备注腔

## 主题系统
- 6套：3日间 + 3夜间
- 日间：奶黄（默认）、粉色、蓝色
- 夜间：对应三套夜间主题
- 手动切换，切换结果持久化
- `theme.css` 只放变量槽位 + 切换机制，不存具体色值
- `data/theme-presets.js` 存具体色值，唯一来源
- `core/theme.js` 负责把色值应用到变量槽位
- 所有页面走同一套 CSS 变量
- 字体粗体，颜色棕黑/灰黑/奶白/米白，适配主题

## 角色系统
- 所有角色数据隔离（记忆、聊天、资料、状态）
- 切换角色 = 切换作用域
- AI 记忆第一人称，符合角色人设

## 工作流
1. 接到任务 → 先看现有代码 → 判断影响范围
2. 改代码 → 自检 → 确认联动没断 → 确认没牵连其他APP
3. 遇到不确定的 → 问用户，用大白话说明选项和后果
4. 每阶段完成后核对 roadmap

## 文件拆分规则
- 超过 800 行必须拆成子文件
- 拆文件不砍功能，不阉割逻辑
- 按单一职责拆分，调用关系自洽
- 用户找问题 → 找对应文件夹 → 只改对应文件
