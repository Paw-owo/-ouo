# UI审美规范

## 做什么
这份规则管整个小手机系统的视觉审美方向。

不管你在做哪个APP，只要涉及界面外观、配色、组件样式、动画、图标、弹窗、卡片，就必须遵守这份规则。

---

## 边界说明
这份skill只定义视觉方向，不单独改动APP结构、信息架构、功能归属和联动机制。

涉及桌面专属布局、桌面交互、APP入口组织时，优先服从 `desktop` skill；这份skill只约束其视觉呈现。

这份skill可以约束设置项的视觉组织方式，但不定义设置项应该归属于全局设置、APP内设置还是会话设置；归属边界以 `settings-center` 和 `app-internal-settings-rules` 为准。

涉及消息页的信息架构、聊天模式、输入区能力、会话设置层级时，优先服从 `message-app`；这份skill只约束其视觉呈现。

优先沿用现有项目的变量体系、组件习惯和已确认视觉方向，不为了套用这份规则而另起一套UI系统。

---

## 这是方向约束，不是死模板
这份规则给的是审美方向和底线，不是让AI照抄CSS片段。

AI应该在遵守方向的前提下，根据当前APP的实际情况举一反三、自由发挥。

---

## 必须遵守的底线
- 所有颜色用CSS变量，禁止硬编码有色值的颜色表达
- 允许使用 `transparent`、`currentColor`、`inherit`
- 优先沿用项目现有 `theme.css` 中已经存在的变量命名，不擅自新建平行变量体系
- 阴影用同色系半透明，禁止灰色 `rgba(0,0,0,x)`
- 文字不用纯黑 `#000`
- 图标本身优先SVG线条风；若使用Stitch布料质感图标底盘，图标内容可以是SVG或图片，但底盘结构必须走变量体系
- 大圆角，禁止直角容器
- 动效轻柔弹性，禁止 `linear`，禁止超过350ms
- 字体用中文无衬线，字重400-600

---

## 允许发挥的空间
- 在遵守变量体系的前提下，可以自由组合渐变、阴影、圆角
- 可以根据APP类型自己设计组件变体
- 可以创意发挥动画细节，只要风格统一
- 可以自己判断什么场景该用什么组件形态
- 可以举一反三补出规则没列的组件

---

## 不允许的事
- 不要因为规则里没列某个组件就不会做
- 不要只会抄规则里的写法，不会根据场景变通
- 不要为了“安全”把所有组件做成一模一样
- 不要因为留了创意空间就乱发挥到风格断裂
- 不要为了套本规则而推翻现有已确认的界面结构和变量体系

---

## 风格定义
**风格名称：** Soft Cozy Minimal（温柔软萌极简风）

**核心关键词：** 软萌 / 干净 / 精致 / 轻盈 / 不塑料 / 像在呼吸 / 可以摸

**氛围方向：**
- 棉花糖甜，不是硬糖甜
- 像一首轻柔的钢琴曲，安静但有温度
- 像精致少女文具店，不是卡通玩具店
- 柔和但有质感，不是廉价塑料感
- 有布料的触感，不是玻璃或金属的冷感

**2026趋势对齐方向（可选吸收，不是必须照搬）：**
- Soft-tech pastels：柔和科技感粉彩，迷雾薰衣草、水洗粉、柔和紫
- Neumorphism软UI：双向柔和阴影，元素像从背景里浮出来
- Claymorphism黏土风：3D圆润感，柔和阴影 + 柔和色彩
- Stitch布料质感：格纹底盘、缝线边框、奶白垫片，有手工感但精致
- Soft 3D icons：圆润、柔软、有触感
- 大圆角软边缘：温暖、亲和、人性化

AI可以从中选取和当前项目风格契合的元素吸收，不需要全部用上。

## 配色体系

### 变量使用铁律
所有颜色必须用CSS变量。组件里禁止出现任何硬编码有色值的颜色表达。

包括但不限于：背景、文字、边框、阴影、遮罩、角标、毛玻璃、渐变、分割线、图标底盘、缝线颜色、装饰元素。

允许使用 `transparent`、`currentColor`、`inherit` 这类不承载具体有色值的表达，但只要涉及实际颜色取值，必须统一走变量。

文档中的色值示例仅用于说明审美方向或变量职责，不是让你直接写进组件代码。实际落地时，颜色必须统一收敛到 `theme.css` 的变量值中；组件层只允许引用变量，不允许直接写 `#hex`、`rgb()`、`rgba()` 等有色值表达。

### 变量命名规范
优先沿用项目现有 `theme.css` 中已经存在的变量，不擅自新建平行命名。

当前已确认的核心变量包括：

- `--bg-base`
- `--surface`
- `--hover`
- `--color-primary`
- `--color-primary-light`
- `--color-primary-ultralight`
- `--color-primary-deep`
- `--color-accent`
- `--color-accent-light`
- `--text-primary`
- `--text-secondary`
- `--text-placeholder`
- `--shadow-soft`
- `--shadow-card`
- `--shadow-float`
- `--shadow-neu-out`
- `--shadow-neu-in`
- `--radius-sm`
- `--radius-md`
- `--radius-lg`
- `--radius-xl`
- `--radius-full`
- `--ease-soft`
- `--ease-smooth`
- `--duration-fast`
- `--duration-normal`
- `--color-success`
- `--color-warning`
- `--color-error`
- `--color-info`

```css
/* 图标底盘 / Stitch质感专用变量（仅在当前任务确实需要时增量补充到 theme.css 主题层） */
--icon-bg;                /* 图标格纹底盘主色 */
--icon-grid;              /* 格纹线颜色 */
--icon-stitch;            /* 缝线虚线颜色 */
--icon-inner;             /* 图标奶白垫片背景 */
--icon-shadow;            /* 图标底盘主投影色 */
--icon-shadow-near;       /* 图标近投影色 */
--icon-shadow-inset-top;  /* 顶部内光 */
--icon-shadow-inset-btm;  /* 底部内阴 */
--icon-deco;              /* 小蝴蝶 / 星星 / 花朵装饰色 */

/* 装饰性小元素 */
--deco-primary;           /* 主装饰色 */
--deco-secondary;         /* 辅助装饰色 */
--texture-dot;            /* 背景点阵 / 纹理颜色 */
```

如果当前任务要扩展变量，只能在不破坏现有体系的前提下谨慎补充，并保持单一来源，不得平行复制一套相近含义的新变量。

如果当前任务并不需要Stitch图标底盘、装饰纹理或相关高级视觉效果，就不要为了“规则完整”硬加整套新变量。只有确实使用到这些视觉模块时，才允许把相关变量增量补进 `theme.css`，且必须放在全局主题层统一管理，不能在组件文件里私建局部变量体系。

### 配色风格要求
不限定具体色值，但必须满足：
- 整体柔和低饱和，禁止荧光色或高对比刺眼色
- 背景极淡浅色，接近白色但有温度感（奶白 / 米白 / 浅粉 / 浅紫）
- 阴影颜色与主色系同色系，禁止灰色阴影
- 文字深色但不用纯黑
- 功能色（成功 / 警告 / 错误）用低饱和柔和版本

### 配色结构建议
推荐“两淡一中一深”的四色结构：
- 两个极淡色做背景层和卡片层
- 一个中间调用作主色或辅助色
- 一个深色做文字或强调

AI可以根据主题自由搭配，但结构要清晰有层次。

### 主题方向（不限于此）
- 奶黄色系：温暖柔和，默认主题
- 粉色系：甜美温柔
- 蓝色系：清新安静

薰衣草紫、奶茶棕等方向只可作为局部审美灵感、插画氛围或未来扩展参考，不代表当前项目要新增正式主题系统。未被当前任务明确要求时，仍只维护现有正式三套主题：`honey` / `pink` / `sky`。不要擅自扩展第四套、第五套主题，也不要顺手修改现有主题切换逻辑。

每个主题都走同一套变量名，切换主题只改变量值，不改组件代码。

图标底盘的格纹色、缝线色、投影色也随主题变量切换，不写死颜色。

## 卡片设计

### 基础卡片
- 白底或极淡主色背景
- 大圆角（`--radius-lg` 或 `--radius-xl`）
- 柔和同色系阴影
- 边框可用极淡主色描边
- 内边距充足，不要挤

### Neumorphism软卡片
适合桌面小组件、图标底盘、轻交互面板等场景：

- 卡片背景和页面背景颜色接近
- 使用柔和双向阴影营造轻微凸起感
- 凸起感要柔和，不能硬
- 点击时可以切换成内凹反馈或其他同风格轻反馈

### 强调卡片
- 可用极淡主色渐变背景
- 适合用在需要突出的内容区域
- 渐变要极其淡，不能抢内容

### 卡片创意空间
AI可以根据场景自由设计卡片变体，例如：
- 带顶部色条的卡片
- 带角落装饰的卡片
- 带渐变光晕的卡片
- 带毛玻璃叠加的卡片
- 带极淡背景纹理的卡片

只要风格统一、变量合规、圆角柔和，就可以发挥。

## 按钮设计

### 主按钮
- 主色渐变填充
- 胶囊形（`--radius-full`）
- 点击有轻微缩放反馈，例如 `scale(0.97)` 一类方案
- 阴影用主色系半透明
- 过渡用弹性缓动

### 次级按钮
- 白底或 `surface` 底
- 主色系细边框
- 轻微凸起感阴影
- 点击可切换成内凹阴影或其他同风格按压反馈

### 图标按钮
- 圆形或圆角足够大的胶囊形
- 最小点击尺寸不小于 `44 × 44`
- 图标用线条风SVG
- 点击有清晰但轻柔的反馈

### 按钮创意空间
AI可以自由设计按钮变体，例如：
- 带图标的按钮
- 带渐变光晕的按钮
- 带加载动画的按钮
- 带确认态的按钮

只要风格统一、变量合规、反馈清晰，就可以发挥。

## 底部弹窗 / 小抽屉（Bottom Sheet）

### 适用场景
- 选项列表
- 快捷操作菜单
- 配置面板
- 选择器
- 确认操作

### 视觉方向
- 从底部滑入
- 推荐使用毛玻璃或柔和半透明面板方案
- 顶部大圆角（`--radius-xl`）
- 顶部有小把手（drag handle）
- 遮罩半透明
- 弹性缓动滑入

### 交互方向
- 点击遮罩关闭
- 下滑关闭
- 有明确关闭按钮或把手
- 内容多时可滚动
- 按钮在拇指可及范围

### 创意空间
AI可以自由设计弹窗变体，例如：
- 带标题和描述的弹窗
- 分段式弹窗
- 带选项卡切换的弹窗
- 带搜索框的弹窗

只要风格统一、交互清晰、不困住用户，就可以发挥。

## 折叠与选项组织

### 选项组织原则
当某个页面选项过多时，不要全部平铺堆在一起。

按以下优先级处理：
1. 高频项放一级页面直接可见
2. 低频项折叠收起，点击展开，标题右侧有清晰提示
3. 关联性强的项归到二级页面，一级只放入口
4. 临时操作类项用底部小抽屉，不占主页面位置
5. 危险操作（删除、清空、重置）单独放最底部，加警示文案，不能和普通设置混在一起

### 折叠区设计方向
- 标题清晰，右侧有展开 / 收起指示
- 展开动画可用 `max-height`、`opacity`、`translateY` 等轻柔方案组合实现
- 展开后内容缩进对齐
- 不要嵌套太多层折叠，最多两层

### 创意空间
AI可以自由设计选项组织方式，例如：
- 分组卡片
- 步骤式
- 标签页切换
- 手风琴

只要不把所有选项平铺成超长列表、层次清晰、用户找得到，就可以发挥。

## 动画效果

### 缓动函数
- 弹性缓动：如 `cubic-bezier(0.34, 1.56, 0.64, 1)` 一类柔和回弹方案，用于弹窗、卡片出现、按钮反馈
- 平滑缓动：如 `cubic-bezier(0.4, 0, 0.2, 1)` 一类平滑过渡方案，用于颜色变化、透明度变化
- 禁止 `linear`

### 时长范围
- 点击反馈：约200ms
- 普通过渡：约280ms
- 弹窗出现：约300~320ms
- 不超过350ms

### 常用动画方向
- 弹入：透明度、轻微缩放、轻微位移组合
- 滑入：透明度、纵向位移组合
- 呼吸：轻微透明度与缩放循环，可用于提示态
- 点击：轻微缩放反馈
- 淡出：用于关闭和消失

### 创意空间
AI可以自由设计动画细节，例如：
- 带轻微旋转的弹入
- 带轻微模糊的淡入
- 带弹性回弹的滑入
- 带错位延迟的列表项依次出现

只要风格轻柔、不炫技、时长在范围内、缓动有弹性，就可以发挥。

## 微动效细节
这一章定义“有生命感”的细节交互。这些微动效让界面感觉在呼吸、在回应用户，是Soft Cozy风格的重要组成部分。

### 点击涟漪
- 点击时在点击位置产生极淡的圆形扩散波纹
- 用伪元素 + `scale` + `opacity` 实现，不用第三方库
- 波纹颜色走 `--color-primary` 或 `--icon-shadow`
- 扩散时长约280ms，淡出约200ms

### 输入框聚焦反馈
- `focus` 时边框变主色 + 极淡主色外光晕
- 可加轻微 `translateY(-1px)` 上浮感，不超过2px
- 配合 `--ease-soft` 弹性缓动，约200ms

### 成功 / 完成操作粒子
- 操作成功时可选加：小圆点从中心 burst 扩散后淡出
- 粒子颜色走 `--color-success` 或 `--deco-primary`
- 粒子数量3~6个，不要太多
- 整个动画约320ms，不超过350ms

### 开关 / Toggle弹性
- thumb移动走弹性缓动，有轻微过冲回弹感
- track颜色渐变切换，不跳变

### 图标底盘点击反馈
- `scale(0.93)` + 阴影同步收紧
- 松开时弹性回弹到原始大小
- 走 `--ease-soft` 缓动

### 列表项出现
- 多个列表项依次出现时，加 `animation-delay` 错位延迟
- 每项延迟约30~50ms，整体节奏轻快不拖沓
- 单项动画约200ms

### 微动效使用原则
- 微动效是点缀，不是主角；每个页面不要超过2~3种不同的微动效
- 不炫技，不堆砌，用在真正有反馈意义的地方
- 所有微动效都要尊重 `prefers-reduced-motion`，减动模式下降级到无动画或简单淡入淡出

## SVG图标画法

### 基础规则
- `viewBox="0 0 24 24"` 标准画布
- `fill="none"`，线条风不填充
- `stroke="currentColor"`，颜色跟随父级
- `stroke-width` 推荐在 `2` 到 `2.5` 之间，走粗线条可爱风
- `stroke-linecap="round"` 必须加
- `stroke-linejoin="round"` 必须加

### 风格关键词
圆润、粗线条、软萌、饱满、一眼可辨

### 禁止
- 细线条针尖图标（`stroke-width` 低于 `1.5`）
- 填充实心图标
- 尖锐端点和转角
- 复杂到缩小后看不清
- emoji代替图标

### 图标在底盘内占比
- 默认75%左右
- 细线条图标不低于72%
- 粗线条图标可到78%
- 禁止低于66%
- 要一眼看清、饱满可爱

### 创意空间
AI可以自由设计图标形状，例如：
- 动物脸（猫、熊、兔）
- 自然元素（月亮、星星、云、花）
- 生活物件（杯子、书本、音符、相机）
- 抽象图形（爱心、爪印、笑脸）

只要线条圆润粗壮、一眼可辨、风格统一，就可以发挥。

### 图标颜色
- 走CSS变量，优先用白色或主色系线条
- 禁止纯黑
- 所有图标统一色系，禁止每个图标不同颜色
- APP区分靠线条形状、名称、布局，不靠不同硬编码底色

## Stitch布料质感图标
这是小手机图标底盘的核心视觉方向。底盘有“可以摸”的布料质感——格纹底、缝线边、奶白垫片三层结构。颜色全部走变量，主题切换时图标底盘颜色同步跟随。

这套方向适用于当前任务明确要做布料质感图标底盘、桌面图标视觉升级、技能图标容器升级等场景；如果当前任务并未要求这类底盘，就不要强行把所有图标都改成Stitch结构。

### 底盘结构（三层）

#### 第一层：奶格纹底盘
- 底色走 `--icon-bg`
- 格纹用双向 `repeating-linear-gradient` 叠加，线颜色走 `--icon-grid`
- 格子间距建议12~14px
- 大圆角，走 `--radius-xl`

#### 第二层：缝线边框（伪元素 `::before`）
- `position: absolute; inset: 6~8px`
- `border: 1.5~2px dashed var(--icon-stitch)`
- `border-radius` 比外层小约8px，保持内缩感
- 颜色走 `--icon-stitch`

#### 第三层：奶白垫片
- `position: absolute; inset: 16~20px`
- 背景走 `--icon-inner`
- 大圆角
- 轻微内光晕
- SVG图标或图片放置在垫片内居中

### 3D浮起感
用多层 `box-shadow` 叠加实现，全部走变量：
- 顶部内光：`--icon-shadow-inset-top`
- 底部内阴：`--icon-shadow-inset-btm`
- 主投影：`--icon-shadow`
- 近投影：`--icon-shadow-near`

其中 `--icon-shadow` 系列变量走主色系半透明，禁止灰色。

### 点击反馈
- `:active { transform: scale(0.93); }`
- 阴影同步收紧
- 过渡走 `--ease-soft` / `--ease-smooth`

### 小装饰元素（角落）
- 可在图标底盘角落放极小的装饰SVG（蝴蝶 / 星星 / 小花 / 爪印）
- 尺寸约10~12px
- 颜色走 `--icon-deco`
- 透明度0.6~0.8，不抢图标主体
- 位置：左上角或右下角，不遮挡图标内容
- 禁止用emoji做装饰

### 图标内容兼容性
图标内容可以是：
- SVG（走 `currentColor` 跟随主题）
- 图片（PNG/WebP）

用图片时：
- 图片本身颜色不随主题变，但底盘颜色仍随主题切换

用SVG时：
- `stroke` 或 `fill` 走变量，保证对比度

### 垫片内图标占比
- 不低于66%
- 推荐75%

### 主题适配
每套主题只需修改变量值，图标底盘代码不动。

下面这些色值块仅用于说明“变量职责怎么分”，不是让你直接复制进组件代码。真正落地时，应把对应值收进 `theme.css` 的主题变量定义里，并继续沿用当前正式三套主题 `honey` / `pink` / `sky`，不要顺手扩展新主题系统。

## 装饰性小元素
这一章定义可以出现在背景或组件角落的装饰元素，用于增加界面温度感，但不抢主内容。

### 背景纹理
- 极淡点阵或圆点纹理，透明度2%~5%
- 颜色走 `--texture-dot`
- 用 `radial-gradient` 或 SVG pattern 实现，不用图片
- 只在大面积背景区域使用，不在卡片或弹窗内滥用

示例（圆点底纹）：
```css
background-image: radial-gradient(circle, var(--texture-dot) 1px, transparent 1px);
background-size: 20px 20px;
```

### 角落小装饰
- SVG用于卡片角落、图标底盘角落、页面空白区域
- 形态：小蝴蝶、小星星、小花朵、小爱心、小爪印
- 必须是SVG，禁止emoji
- 颜色走 `--deco-primary` 或 `--deco-secondary`
- 透明度0.5~0.7，不抢主体
- 尺寸约10~16px，不要大
- 每个组件最多1~2个装饰，不要堆砌

### 渐变光晕
- 可在卡片或页面区块角落加极淡的径向渐变光晕
- 颜色走 `--color-primary-ultralight` 或 `--color-accent-light`
- 透明度20%以内
- 增加空间层次感，不做主视觉

### 分割装饰
- 分割线可以用带小装饰的中心断开式
- 左右线条 + 中间小星星 / 花朵元素
- 颜色走 `--deco-primary`
- 比纯分割线更有温度，适合个性化页面

### 使用原则
- 装饰是点缀，不是主角
- 克制使用，每个页面装饰元素总量控制在3~5处以内
- 优先考虑去掉装饰是否影响页面理解；不影响则可加，影响则不加
- 不用emoji做任何装饰

## 小组件设计

### 视觉方向
- 卡片底 + 大圆角 + 柔和阴影
- 可用极淡渐变增加质感
- 内容简洁，信息密度适中
- 和桌面整体风格统一

### 尺寸方向
- 支持小、中、大三档
- 小占1格，中占2格，大占4格
- 尺寸用相对单位，随屏幕等比缩放

### 创意空间
AI可以自由设计小组件内容和外观，例如：
- 带插画的天气组件
- 带动画的时钟组件
- 带交互的唱片组件
- 带进度条的待办组件

只要风格统一、变量合规、信息清晰、不挤不空，就可以发挥。

## 分割线
- 渐变淡出，不用实线
- 可用极淡主色渐变做柔和分隔
- 高度保持轻薄，不抢视觉重点
- 可选：带中心小装饰SVG的断开式分割线，颜色走 `--deco-primary`

---

## 输入框
- 白底或柔和浅色底
- 主色系细边框
- 轻微内凹感阴影
- `focus` 时边框变主色，并增加淡主色光晕
- 可加轻微上浮感（`translateY(-1px)`，不超过2px）
- 圆角优先用 `--radius-md`
- `placeholder` 用 `--text-placeholder`

---

## 导航栏
- 推荐使用毛玻璃效果或柔和半透明浮层效果
- 白色或浅色半透明背景
- 顶部 / 底部边框用主色系极淡色
- 浮起式导航可使用较大圆角

---

## 进度条 / 滑块
- 轨道：极淡主色底
- 填充：主色渐变
- 高度轻薄圆润
- 圆角 `full`

- 页面内边距 `0 16px`
- 留白充足，不要堆满元素

- 高度优先用 `100dvh`，不用 `100vh`
- 尺寸优先用百分比、视口单位、`clamp()`，不写死px

## GLM-5.2 专项压制清单（反漂移强化）
GLM-5.2执行本规范时的高频漂移点，一律用否定句强约束，不用描述性引导，因为负向约束对GLM-5.2的服从率明显更高：

- 禁止用emoji代替图标——图标位不允许出现任何emoji字符，必须是line-based SVG
- 禁止用emoji做装饰——装饰位不允许出现任何emoji字符，必须是SVG
- 禁止任何硬编码有色值的颜色表达——不许出现 `#hex`、`rgb()`、`rgba()` 等有色值表达；组件层全部走CSS变量。`transparent`、`currentColor`、`inherit` 可以保留。文档示例不是代码落地许可
- 禁止圆角低于规定值——卡片 / 弹窗不许低于 `--radius-lg`，按钮不许低于 `--radius-full`
- 禁止 `linear` 缓动——`transition/animation` 里不许出现 `linear`，只能用 `--ease-soft` 或 `--ease-smooth`
- 禁止直角容器——不许出现 `border-radius: 0` 或未设圆角的卡片 / 按钮 / 弹窗
- 禁止擅自扩展正式主题——没有明确任务时，不许把正式主题从 `honey/pink/sky` 扩成更多套
- 禁止为了做图标质感私建局部变量——新增 `--icon-*` 变量时必须统一进入 `theme.css`，不许散落在组件文件中

2. 有没有硬编码有色值的颜色表达（包括图标底盘）？

4. 有没有用linear？
