# 小手机 UI 审美规范
> 本文件用于指导 AI 进行界面设计与美化。
> **配色通过全局 CSS 变量控制，AI 不得硬编码色值。风格规范必须严格遵守。**

---

## 一、风格定义

**风格名称**：Soft Cozy Minimal（温柔软萌极简风）

**核心关键词**：软萌 / 干净 / 精致 / 轻盈 / 不塑料 / 像在呼吸

**氛围描述**：
- 不是硬糖甜，是棉花糖甜
- 像一首轻柔的钢琴曲，安静但有温度
- 像精致少女文具店，不是卡通玩具店

---

## 二、配色系统

### 使用方式（重要）
**所有颜色必须使用 CSS 变量，不得在组件中硬编码色值。**

### 配色风格要求
- 整体色调必须**柔和低饱和**，不能出现荧光色或高对比刺眼色
- 背景色必须是**极淡的浅色**，接近白色但有温度感（奶白/米白/浅粉/浅紫都可）
- 阴影颜色必须与主色系**同色系**，禁止使用纯灰色阴影
- 文字颜色用**深色调**，但不用纯黑 `#000000`
- 功能色（成功/警告/错误）用**低饱和柔和版本**

---

## 三、形状规范

--radius-sm: 10px / --radius-md: 16px / --radius-lg: 20px / --radius-xl: 28px / --radius-full: 999px

**原则：宁可太圆，不要有棱角感。所有可见容器必须有圆角，禁止直角矩形。**

---

## 四、阴影规范

- 所有阴影必须**柔和扩散**，不允许硬边阴影
- 阴影颜色必须是**当前主色系的半透明色**，不用 `rgba(0,0,0,x)`
- 卡片默认有轻微阴影，有层次感但不抢眼
- 需要凸起感的按钮/图标底盘使用 neumorphism 双向阴影

---

## 五、动效规范

- 基础过渡：`cubic-bezier(0.34, 1.56, 0.64, 1)`，时长 200-300ms
- 点击反馈：`scale(0.97)`
- 禁止 linear 缓动

---

## 六、禁止事项

- 硬编码任何色值
- 使用高饱和荧光色
- 使用灰色系阴影 `rgba(0,0,0,x)`
- 使用纯黑 `#000000` 作为文字色
- 使用 emoji 代替图标
- 使用填充实心图标
- 使用直角容器（无圆角）
- 使用衬线字体
